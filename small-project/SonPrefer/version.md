# version.md

## Scope

### [v1.0.9] - 2026-07-22
- **결과 테이블 상세 정수 복구**: UI 결과 테이블의 총 출생아 수는 겹침 우려가 없으므로 축약하지 않고, 원래의 상세 정수(천 단위 콤마 포맷인 `formatCommas`) 형식으로 표기되도록 복원했습니다.
- **차트 콤팩트 소수점 분기 처리**: 그래프 및 Y축 눈금선 수치에 대해 만 단위는 소수점 1자리(예: `1600.0만`), 억 단위는 소수점 2자리(예: `10.50억`), 만 미만은 정수(예: `400`) 형태로 각각 가독 단위의 소수점 자릿수를 분기 조율했습니다.

### [v1.0.8] - 2026-07-22
- **한국식 콤팩트 만/억 단위 포맷터 설계**: 차트 수치 및 결과 테이블 수치 표현 시 `1600.00만`, `1.60억` 형태로 만/억 단위를 적용하고 고정 소수점 2자리(`.toFixed(2)`)를 고수하여 글자 겹침을 방지하고 레이아웃 요동을 차단했습니다.
- **Monospace 서체 강제**: 결과 텍스트 인풋 및 SVG 텍스트 태그에 고정 자폭(`monospace`) 스타일을 적용해 가변 너비로 인한 흔들림 현상을 완전해소했습니다.
- **LaTeX KaTeX 엔진 통합**: 챕터2 내 수학 수식(`\( \frac{1}{p} \)`)이 일반 문자열로 노출되던 문제를 해결하고자 KaTeX 라이브러리 CDN을 탑재하고 `triggerMathRendering()`을 통해 자동 렌더링되게 개선했습니다.

### [v1.0.7] - 2026-07-22
- **완료 시 이중 애니메이션(리셋) 제거**: 실시간 렌더링을 통해 100%까지 채운 그래프가 연산 완료 순간에 0으로 회귀하여 다시 애니메이션되던 이중 렌더링 버그를 수정했습니다. 연산 완료(`complete`) 시점에 `redrawChart()` 대신 `updateChartRealtime()`을 곧바로 호출해 즉각 최종 화면을 고정 정렬합니다.

### [v1.0.6] - 2026-07-22
- **Y축 스케일 사전 고정 (흔들림 버그 패치)**: 시뮬레이션 가동 직전에 전체 가구 수를 바탕으로 기댓값의 이론적 상한선(`householdsCount * 1.3`)을 구해 Y축 최댓값으로 단 한번 고정하도록 개선했습니다. 이로써 실시간 렌더링 중 Y축 눈금선이 춤추듯 요동치지 않고 고정된 차트 좌표 내에서 막대 그래프만 차분히 차오르도록 시각적 정합성을 완성했습니다.

### [v1.0.5] - 2026-07-22
- **인라인 Blob Web Worker 설계 전환**: 부모 SPA 주입 호스팅 환경에서 정적 웹 워커 스크립트 파일의 상대 경로 탐색 실패로 발생하는 404 로드 에러(워커가 0%에서 멈추던 버그)를 메인 스레드 내 소스코드 문자열 상수 내장 및 Blob URL 동적 생성 구조로 극복하여 원천 차단했습니다.

### [v1.0.4] - 2026-07-22
- **실시간 진척도(%) 스트리밍**: Web Worker 가 루프 연산 도중 1% 단위(가구수에 비례한 주기)로 현재의 계산 진행도를 전송하고, 메인 스레드는 이를 스피너 영역에 `N%` 형태로 실시간 노출하도록 확장했습니다.
- **실시간 차트 빌드 & 성장 연출**: 연산 완료까지 멈춰 있던 막대 차트를 개선하여, 워커로부터 실시간으로 갱신되는 중간 집계 데이터를 받는 즉시 화면의 SVG 막대가 역동적으로 차올라 자라나는 실시간 그래픽 렌더러(`updateChartRealtime`)를 구현했습니다.

- **UI 및 명칭 정비**:

- **양방향 슬라이더(Range) 연동**: 가구수 및 자연성비 입력 필드에 슬라이더 입력을 추가하고, 숫자 텍스트 창과 슬라이더가 상호 동기화되는 양방향 바인딩을 이식했습니다. (자연성비 범위: 80~120)
- **자바스크립트 버그 패치 및 호환성 강화**:

  - 호스트 웹사이트 동적 주입 시 발생하던 `DOMContentLoaded` 무반응 문제를 해결하기 위해 `checkAndInit()` 우회 수동 초기화 모듈을 이식했습니다.

### [v1.0.0] - 2026-07-22
- **시작 페이지 전환**: 기존 `SonPrefer.html`을 `index.html`로 이관 및 대체하였습니다.
- **표준 모듈화 설계 적용**:
  - `config.js`: 상수 설정 및 상태(state) 정보 통합 관리.
  - `simulation.js`: 시뮬레이션 계산 로직을 분리하고 DOM 제어를 완전히 배제.
  - `chart.js`: SVG 렌더링, 반응형 그리드라인 계산 및 차트 애니메이션 제어.
  - `main.js`: DOM 이벤트 연결, 시뮬레이션 트리거 및 생명주기 관리.
- **CSS 격리 및 호환성 개선**: `style.css`를 `SonPrefer.css`로 이관하며 `body` 및 `article`에 설정되었던 고정 레이아웃 스타일(width, max-width, margin 등)을 배제하여 부모 SPA 웹사이트와의 레이아웃 호환성을 보장하도록 개선했습니다.
- **시뮬레이션 논리 오류 수정**: 남아를 출생하면 출산을 중단해야 하나 기존 조건식에서 여아가 나올 때까지 계속 낳는 오작동(`while(birth < ratio)`)을 `while(birth >= ratio)`로 수정하여 원래 의도대로 남아선호사상이 정확하게 시뮬레이션되도록 교정했습니다.
- **디버깅 에러 콘솔 탑재**: `window.onerror` 감지를 연동해 상단 `#error-console` 영역에 실시간 런타임 오류를 노출하여 개발 편의성을 높였습니다.
